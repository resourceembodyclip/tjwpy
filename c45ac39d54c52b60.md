# WebLink Navigator

WebLink Navigator 是一个面向技术研究人员、内容聚合者与信息分析人员的高密度外链资源归集与导航系统。本项目的核心定位并非提供原创内容，而是针对特定领域内的分散式信息节点进行系统化采集、结构化归档与持久化引用，从而构建一个可复用、可追溯、可扩展的技术参考索引库。

本项目主要服务于以下用户群体：需要进行大规模信息源梳理的技术调研团队、从事移动端内容生态分析的数据采集工程师、以及需要维护长期外链资产清单的文档管理人员。通过将零散的 URL 资源按照语义类别进行归并，本项目显著降低了信息碎片化带来的检索成本，并提供了统一的访问入口与状态监控能力。

本项目已收录第 28/56 批次资源，当前批次包含 180 个有效外链条目，覆盖移动端技术文档、行业资讯、案例分析与技术博客等多元维度。所有资源均保持原始来源的协议、域名与路径结构，确保引用的准确性与可复现性。

## 功能概览

- 批量链接归集：支持一次性导入大量异构 URL，自动完成去重、格式校验与协议一致性检查，生成标准化的资源清单。

- 分类导航视图：根据 URL 的域名特征、路径层级与内容类型自动打标签，提供按主题、按来源、按时间维度的多级筛选与排序功能。

- 链接状态监测：周期性探测每个外链的可达性、响应状态码与页面标题变化，标记失效链接与内容迁移事件，辅助维护人员及时更新。

- 全文元数据提取：对于可访问的 HTML 页面，自动抽取标题、描述、关键词与主要文本摘要，形成本地缓存索引，支持离线检索。

- 自定义标签体系：允许用户为任意链接添加自由标签、备注说明与重要性评级，构建符合个人或团队工作流的分类逻辑。

- 导出与集成接口：支持将资源列表导出为 JSON、CSV、Markdown 表格等多种格式，并提供 RESTful API 便于与其他内部系统对接。

- 变更审计日志：记录每次资源增删改操作的时间戳、操作人与变更内容，满足团队协作场景下的审计合规要求。

## 应用场景

场景一：移动端技术文档库建设
技术团队在日常开发中需要频繁查阅散布于多个域名的 SDK 文档、API 参考与示例代码。通过 WebLink Navigator，团队可以将所有相关文档链接统一录入系统，按技术栈、版本号与功能模块分类，新成员入职时即可一键获取完整文档索引。

场景二：行业资讯监控与简报生成
市场分析人员每日需要浏览数十个行业博客、新闻站点与论坛热帖。本项目提供的批量链接导入与元数据提取功能，能够将分散的资讯聚合为本地摘要，辅助快速筛选高价值内容，并定期生成行业简报初稿。

场景三：外链资产迁移与重构审计
当网站改版或域名更换时，大量历史外链可能面临失效风险。运维人员可利用本系统的链接状态监测功能，批量扫描全部外链的可达性，生成失效报告，并依据原始 URL 的路径规律进行批量替换或重定向配置。

场景四：学术研究与参考文献管理
研究人员在进行文献综述或技术调研时，需要长期追踪大量在线资源。本项目支持为每条链接添加引用日期、访问状态与核心摘要，可作为轻量级参考文献管理工具，辅助撰写技术报告或学术论文。

场景五：内容聚合站点的数据源管理
内容聚合类网站或资讯 App 的后台需要维护一套稳定的数据源列表。WebLink Navigator 提供了结构化的导入导出接口，可定期与上游数据源同步，并自动检测源站点的内容更新频率与可用性变化。

## 快速开始

以下命令演示了如何在本地环境完成本项目的克隆、依赖安装与基础服务启动。

```bash
# 克隆项目仓库至本地
git clone https://github.com/your-org/weblink-navigator.git

# 进入项目根目录
cd weblink-navigator

# 安装项目依赖（使用 npm）
npm install

# 启动开发服务器，默认监听 3000 端口
npm run dev
```

启动成功后，访问控制台输出的本地地址即可进入资源管理界面。首次启动时，系统会自动初始化 SQLite 数据库并创建基础表结构。若需使用 MySQL 或 PostgreSQL，请参考 `docs/database.md` 中的配置说明。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.x 或 20.x LTS | 项目运行时环境，建议使用 nvm 管理版本 |
| npm | 9.x 或更高 | 包管理器，用于安装项目依赖 |
| SQLite3 | 3.35 或更高 | 默认嵌入式数据库，无需额外安装 |
| Python | 3.9 或更高 | 仅当启用元数据提取的 NLP 增强模块时需要 |
| curl | 7.68 或更高 | 用于链接状态监测的底层探测工具 |
| git | 2.25 或更高 | 版本控制工具，用于克隆与更新项目 |
| 内存 | 512 MB 及以上 | 运行时内存要求，大规模导入时建议 1 GB |
| 磁盘 | 1 GB 空闲空间 | 用于存储数据库、缓存与日志文件 |

│   │   │   ├── linkController.js     # 链接资源的增删改查接口
│   │   │   ├── tagController.js      # 标签管理接口

│   │   ├── services/                 # 核心服务层，封装数据访问与外部调用
│   │   │   ├── linkService.js        # 链接归集、去重、分页服务
│   │   │   ├── crawlService.js       # 基于 puppeteer 的元数据抓取服务
│   │   │   └── healthService.js      # 批量链接可达性探测服务
│   │   ├── models/                   # 数据模型定义（Sequelize ORM）
│   │   │   ├── Link.js               # 链接主表模型

## 常见问题

问：导入大量链接时页面出现超时或卡顿，应如何处理？

答：当单次导入链接数量超过 500 条时，建议使用命令行脚本 `scripts/importBatch.js` 进行后台导入，而非通过前端页面提交。该脚本支持分批次提交与进度显示，并可自动跳过格式异常的条目。若仍需通过前端导入，请调整前端 `axios` 的超时配置至 120 秒以上。

问：链接状态监测显示大量超时或 DNS 解析失败，是否代表系统异常？

答：不一定。链接状态监测结果受目标服务器响应速度、网络环境与防火墙策略影响。建议首先通过 `curl -v` 命令在部署该系统的服务器上手动测试目标链接，以区分是网络问题还是目标站点本身不可用。系统默认超时时间为 10 秒，若目标站点响应较慢，可在 `config/health.js` 中调高 `timeout` 参数。

问：如何将现有的 SQLite 数据迁移至生产环境的 MySQL 数据库？

答：项目内置了数据迁移脚本 `scripts/migrateToMySQL.js`。执行前需在 `.env` 文件中配置 `MYSQL_HOST`、`MYSQL_USER`、`MYSQL_PASSWORD` 与 `MYSQL_DATABASE` 变量，并确保 MySQL 版本为 5.7 或更高。迁移脚本会自动创建表结构并逐条复制数据，建议在低峰期执行并提前备份 SQLite 文件。
