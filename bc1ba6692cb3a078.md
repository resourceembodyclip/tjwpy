# WebFront Navigator

WebFront Navigator 是一个面向前端开发工程师与技术决策者的高质量技术资源聚合与导航系统。该项目通过对数百个经过人工筛选的技术文章、工程实践与架构案例进行结构化整理与分类索引，帮助开发团队在技术选型、性能优化、工程化建设与团队规范制定等场景下快速定位到具备实际参考价值的权威资料。

与传统的书签管理工具或散落的收藏夹不同，WebFront Navigator 维护了一套具备清晰主题分类、内容摘要与标签体系的外部资源映射表。项目本身不存储文章内容，而是专注于构建一个高可用、可检索、可扩展的外部知识索引层。目标用户包括需要系统化提升团队技术视野的架构师、负责技术调研与选型的核心开发者，以及希望建立个人知识管理体系的初中级前端工程师。

项目当前处于第 23 批资源收录阶段，累计收录外部技术链接超过 180 条，覆盖从基础原理到前沿实践的多个技术维度。所有链接均经过可用性校验与内容质量初筛，确保资源的有效性与时效性。

## 功能概览

**结构化资源索引** 按照技术领域与内容类型对收录链接进行多级分类，支持按主题、难度层级与适用场景快速筛选。

**内容摘要自动生成** 对每个收录的 URL 通过元信息解析与规则匹配自动生成内容摘要与关键词标签，降低人工维护成本。

**多维度检索查询** 提供基于标题关键词、分类标签与内容特征码的组合检索能力，支持模糊匹配与精确查找两种模式。

**资源状态监控** 定期对已收录链接进行可达性检查与内容变更检测，自动标记失效或发生重大变化的资源。

**批量导入导出** 支持通过标准格式文本批量新增外部链接，并可将索引数据导出为 JSON、CSV 或 Markdown 表格格式。

**访问热度统计** 记录各资源链接在团队内部的点击频次与引用次数，为内容价值评估提供数据支撑。

**分类体系自定义** 允许用户根据自身技术栈特点创建自定义分类标签与分组规则，实现个性化资源组织。

## 应用场景

技术选型与方案评审。当团队需要在新项目中决定状态管理方案、构建工具或渲染架构时，可通过本导航系统快速检索相关领域的对比分析与实践案例，在较短时间内获得多个可参考的决策依据。

性能优化专项攻坚。面对页面加载缓慢、运行时卡顿或内存泄漏等问题，开发者可以通过性能优化分类下的资源链接，系统性地学习从网络层、渲染层到代码执行层的诊断方法与优化策略。

工程化建设与规范制定。在搭建团队的 CI/CD 流程、代码规范体系或监控告警平台时，导航系统中的工程化与自动化相关资源能够提供从理论框架到落地实现的完整参考链路。

新人培训与知识传递。团队新成员可以通过本系统按主题路径进行系统性自学，快速建立对团队技术体系与业界主流实践的认知框架，缩短上手周期。

技术调研与趋势追踪。在评估是否引入新的框架版本、语言特性或开发范式时，通过趋势分类下的最新资源链接把握技术演进脉络。

## 快速开始

以下命令可在本地环境完成 WebFront Navigator 项目的克隆、依赖安装与开发服务器启动。

```bash
git clone https://github.com/webfront-navigator/webfront-navigator.git
cd webfront-navigator
npm install
npm run dev
```

执行成功后，访问控制台输出的本地服务地址即可进入导航系统主界面。默认情况下，开发服务运行在 5173 端口。如需构建生产版本，请执行 `npm run build`，产物将输出至 dist 目录。

│   │   └── cache.ts                    # 索引数据的多级缓存管理
│   ├── api/                            # RESTful API 路由与控制器
│   │   ├── resources.ts                # 资源增删改查与分页查询接口
│   │   ├── categories.ts               # 分类体系的维护接口
│   │   └── stats.ts                    # 访问统计与热度数据接口
│   ├── ui/                             # 前端用户界面组件
│   │   ├── components/                 # Vue/React 可复用组件
│   │   │   ├── ResourceList.vue        # 资源列表渲染与虚拟滚动

│   │   ├── layouts/                    # 页面布局框架
│   │   └── styles/                     # 全局样式与主题变量
│   ├── db/                             # 数据库模型与迁移脚本
│   │   ├── models/                     # 数据表结构定义（Sequelize/Knex）
│   │   ├── migrations/                 # 增量变更迁移文件
│   │   └── seeders/                    # 初始分类与示例数据填充

│   │   ├── health-check.ts             # 定期链接健康检查任务
│   │   └── sync-task.ts                # 外部数据源同步任务

## 常见问题

问：导航系统中收录的链接无法访问怎么办？

答：项目内置的链接健康检查任务会定期自动检测所有收录 URL 的可达性。当检测到失效链接时，系统会在管理后台标记该资源状态为「异常」并通知维护者。作为临时方案，用户可以通过界面上的「报告失效」按钮主动提交反馈。维护者将在两个工作日内核实并更新链接或移除失效条目。如需立即处理，可手动运行 `npm run check:links` 执行全量校验。

问：如何批量导入我自己的技术收藏链接？

答：项目提供了批量导入脚本 `scripts/import-batch.ts`。你可以准备一个 CSV 或 JSON 格式的文件，包含 url、title、category、tags 等字段，然后执行 `npm run import -- --file ./my-links.json`。导入前系统会自动进行去重校验与格式规范化。详细字段说明与示例文件请参考 docs/maintenance/batch-import.md。

问：导航系统的分类体系能否自定义？

答：可以。用户可通过管理界面或 API 创建、编辑与删除分类标签。系统预置了一套通用分类（如「性能优化」「工程化」「框架原理」「网络协议」等），但允许用户完全覆写或扩展。自定义分类后，资源列表会按照新的分组逻辑重新渲染。注意，删除分类前需将其下的资源迁移至其他分类，否则系统会阻止删除操作以避免数据丢失。
