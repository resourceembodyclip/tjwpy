# TechLink Navigator

TechLink Navigator 是一个面向技术研究者和开发者的外链资源聚合与导航系统。该项目通过对分散在互联网各处的技术文章、文档和案例进行结构化收录与分类索引，帮助用户快速定位特定主题下的高质量外部资源。项目本身不存储任何内容，仅提供链接的整理与元数据标注。

项目定位于技术资源的中转站与检索增强层。目标用户包括：需要快速查阅特定技术实现细节的研发工程师、进行技术选型调研的架构师、撰写技术方案或论文的研究人员，以及需要维护团队知识库的技术管理者。通过统一的条目编号体系与批次管理，TechLink Navigator 使得大规模外链资源的引用、回溯与共享变得可操作、可维护。

## 功能概览

**批量资源收录**：支持按批次导入大量外部链接，每批次最高可处理 200 条资源记录，并自动分配内部编号。

**动态分类索引**：根据链接来源域名、路径特征或人工标注，将资源自动归入技术文档、案例分析、API 参考、性能调优等类别。

**原始链接透传**：所有收录的 URL 均以原始格式原样呈现，不添加任何前缀、后缀或跳转中间页，确保访问路径的确定性。

**条目快速检索**：基于资源编号、标题关键词或来源域名进行全文检索，返回匹配结果并按相关性排序。

**批次状态管理**：记录每批资源的导入时间、条目数量和处理状态，支持多批次并行管理与历史回溯。

**结构化输出**：支持将资源列表导出为 Markdown、JSON 或 CSV 格式，便于集成到其他文档系统或自动化工具链中。

## 应用场景

技术方案调研与选型：当团队需要评估某项新技术或中间件时，可通过 TechLink Navigator 快速检索已收录的评测文章、性能对比报告和官方文档链接，大幅减少信息搜集时间。

技术文档撰写与引用：技术博主或文档维护者在撰写教程、设计文档或 RFC 时，可以直接从项目中引用已收录的外部链接，保证引用的准确性和可追溯性。

团队知识库建设：技术团队可以将项目作为内部知识管理系统的外链数据源，定期导入经过筛选的高质量外部资源，形成统一的知识索引层。

离线资源备份规划：对于需要在内网环境或隔离网络中工作的团队，可通过项目提供的批次链接清单，制定有针对性的离线下载与缓存策略。

## 快速开始

以下步骤帮助您在本地环境中快速启动 TechLink Navigator 并导入第一批资源数据。

```bash
# 克隆项目仓库
git clone https://github.com/techlink-navigator/tln-core.git
cd tln-core

# 安装依赖（使用 Python 3.9+ 和 pip）
pip install -r requirements.txt

# 执行数据库初始化与批次导入（示例批次 38）
python cli.py init-db
python cli.py import-batch --batch 38 --source ./data/batch_38_links.json
python cli.py serve --port 8080
```

上述命令完成后，服务将在本地 8080 端口启动，可通过浏览器访问 http://127.0.0.1:8080 查看已导入的资源列表。

## 项目结构

项目采用分层模块化组织，核心代码、配置、文档与测试目录分离。以下为项目根目录的完整 ASCII 目录树。

```
tln-core/
├── cli.py                      # 命令行入口，包含 init-db / import-batch / serve 等命令
├── config.yaml                 # 主配置文件，定义数据库路径、服务端口与批次目录
├── requirements.txt            # Python 依赖列表，固定版本号
├── README.md                   # 项目说明文档（本文件）
├── LICENSE                     # MIT 许可证全文
├── docs/                       # 文档目录
│   ├── user-guide.md           # 用户手册，面向普通使用者
│   ├── admin-guide.md          # 管理员手册，面向批次维护人员
│   ├── developer-guide.md      # 开发参考，面向贡献者
│   └── architecture.md         # 系统设计说明，含数据流与扩展点
├── src/                        # 核心源代码目录
│   ├── __init__.py             # 包初始化
│   ├── app.py                  # Flask Web 应用主入口
│   ├── database.py             # SQLite 数据库连接与表结构定义
│   ├── models.py               # 资源条目、批次、分类等数据模型
│   ├── importer.py             # 批次导入逻辑，含 JSON/YAML 解析
│   ├── searcher.py             # 基于 SQLite FTS 的全文检索实现
│   └── utils.py                # 通用工具函数（URL 规范化、时间戳处理）
├── tests/                      # 单元测试目录
│   ├── test_database.py        # 数据库操作测试
│   ├── test_importer.py        # 导入流程测试
│   └── test_searcher.py        # 检索功能测试
├── data/                       # 批次数据存放目录
│   ├── batch_38_links.json     # 第 38 批资源链接原始数据
│   └── batch_39_links.json     # 第 39 批资源链接原始数据（示例）
├── static/                     # Web 静态资源
│   ├── css/
│   │   └── style.css           # 界面样式表
│   └── js/
│       └── search.js           # 前端检索交互脚本
└── templates/                  # Jinja2 模板目录
    ├── index.html              # 资源列表首页
    ├── detail.html             # 单条资源详情页
    └── batch.html              # 批次管理页
```

## 常见问题

问：导入链接时提示 "URL 格式无效"，但链接本身可以正常访问，是什么原因？

答：导入器默认对链接进行严格的协议检查，仅接受以 http:// 或 https:// 开头的完整 URL。如果您的链接缺少协议前缀，请补全后再导入。此外，链接中若包含未转义的特殊字符（如空格或中文），也可能触发校验失败，建议使用 URL 编码后再试。

问：如何更新已导入批次的资源元数据，例如修改分类或备注？

答：目前项目不支持直接通过 Web 界面修改已导入条目的元数据。您需要找到对应批次的原始 JSON 或 YAML 文件，在本地修改后使用 `cli.py update-batch --batch <编号> --file <新文件>` 命令进行覆盖更新。请注意，该操作会完全替换该批次的所有条目，请谨慎操作并提前备份。

问：检索结果中某些链接返回 404 状态码，系统是否会主动检测并标记？

答：系统提供了可选的链接可达性检查功能，默认在导入时对每条链接发送 HEAD 请求进行验证。如果检测到 4xx 或 5xx 状态码，会在数据库中标记为 "异常" 状态。您也可以通过 `cli.py check-links --batch <编号>` 手动触发重新检查。对于因网络原因偶尔超时的链接，建议增加超时阈值或配置重试次数。
