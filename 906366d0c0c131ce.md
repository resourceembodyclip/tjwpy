# ResourceLink Collective

ResourceLink Collective 是一个面向技术文档工程师、开发者与知识管理从业者的外部资源聚合与索引系统。该项目并非传统的爬虫或链接仓库，而是一套结构化的外链引用规范体系，用于对分散于多源站点的技术文章、案例研究与规范文档进行统一标识、分类归档与版本追踪。其目标用户包括需要维护大规模外部引用列表的文档团队、构建知识图谱的技术内容管理者，以及希望从既定外链集合中快速定位高价值资料的研究人员。通过建立一致的引用格式、分级标签与状态标记，ResourceLink Collective 将原始 URL 集合转化为可查询、可审计、可扩展的资产清单，有效降低链接失效、重复收录与引用混乱带来的维护成本。

## 功能概览

**统一引用标识**：为每条外链生成唯一内部 ID，并保留原始 URL、标题摘要与收录时间戳，支持跨文档精确引用。

**批量状态检测**：周期性对收录链接进行可达性检查，自动标记失效、重定向或内容变更的条目，输出状态报告。

**多级分类标签**：允许用户为每个链接附加一个主分类与多个子标签，支持按技术领域、内容类型、来源站点等维度筛选。

**元数据扩展字段**：提供可自定义的元数据模板，包括作者、发布日期、关联项目、关键术语等，满足深度检索需求。

**导入导出适配器**：支持从 CSV、JSON、Markdown 表格批量导入链接，并可导出为静态站点导航页或 API 接口数据。

**版本变更日志**：记录每次链接新增、删除、元数据修改的操作历史，支持回滚与审计追溯。

## 应用场景

技术文档团队在发布产品新版本时，需要统一更新所有引用外部规范文档的链接。ResourceLink Collective 的批量状态检测与版本日志功能可快速识别哪些链接已失效或内容过时，并协助团队一次性替换为新版链接，避免文档中出现死链。

知识库管理员定期从多个技术博客、会议演讲和开源仓库收集优质外链，用于构建内部周报或学习路线图。利用多级分类标签与元数据扩展字段，管理员可以将这些链接按主题、难度和来源组织成可浏览的导航树，供团队成员按需查阅。

开源项目维护者在 README 中引用了大量第三方工具和参考资料，随着项目迭代，这些外部链接可能逐渐失效。通过 ResourceLink Collective 生成的外链状态徽章，维护者可以直接将状态信息嵌入 README，让用户清楚知道每个参考链接的当前可用性。

## 快速开始

以下命令演示如何在本地环境部署 ResourceLink Collective 并导入首批链接数据。

```bash
# 克隆项目仓库
git clone https://github.com/resourcelink/collective.git
cd collective

# 安装依赖（使用 pip 或 poetry）
pip install -r requirements.txt

# 运行初始化命令，创建本地数据库与配置文件
python cli.py init --db-path ./data/links.db

# 导入示例链接列表（CSV 格式）
python cli.py import --source ./samples/initial_links.csv --format csv

# 启动本地 Web 查看服务（默认端口 8080）
python cli.py serve --port 8080
```

```
└── README.md              # 本文件
```

## 常见问题

**问：项目是否会主动爬取或缓存链接指向的页面内容？**

答：不会。ResourceLink Collective 仅执行轻量级 HTTP HEAD 请求以检测链接可达性，不会下载或存储页面主体内容。所有操作均遵循 robots.txt 规范，且可配置请求间隔以避免对目标服务器造成压力。

**问：如何迁移已有的大量外链数据到本系统中？**

答：项目内置了 CSV 和 JSON 导入适配器。用户只需将原有链接整理为包含 `url`、`title`、`category` 等列的表格，通过 `cli.py import` 命令即可批量导入。对于来自 Sphinx 项目或 RSS 订阅源的数据，可使用 `plugins/` 中的专用适配器进行转换。

**问：状态检测结果为「内容变更」的依据是什么？**

答：系统会记录每次检测时响应头中的 `Last-Modified` 与 `ETag` 值。当连续两次检测中这两个值发生变化且响应状态仍为 200 时，标记为「内容变更」。该策略可配置为基于响应体哈希值的深度检测，但默认采用头部比较以降低带宽消耗。
