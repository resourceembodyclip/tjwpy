# WebLink Navigator

WebLink Navigator 是一个面向技术研究、信息检索与知识管理场景的轻量级外链资源汇总与导航系统。该项目定位于为开发者、研究人员及内容策展人提供结构化的外部链接托管、分类展示与快速检索能力，解决分散收藏、链接失效、缺乏上下文标注等常见信息管理痛点。

项目采用静态站点生成方案，将外部 URL 与元数据（来源、批次、标签、摘要）组织为可维护的目录结构，支持本地预览与一键部署。通过批次化管理模式（当前为第 52/56 批，共计 180 个资源链接），项目能够承载持续增长的外链库，并保持清晰的版本追溯与审核流程。

## 功能概览

**批次化资源导入**：支持按批次（每批 100-200 条）批量导入外部 URL，自动生成唯一编号与入库时间戳，便于追踪资源来源与更新周期。

**多维度分类导航**：资源可按主题领域、文件类型、来源站点进行自动或手动分类，生成多级筛选视图，提升大规模链接库的浏览效率。

**链接健康状态检测**：内置链接可达性检查工具，定期对已收录 URL 发起 HEAD 请求，标记失效或重定向链接，辅助维护人员及时清理或更新。

**元数据扩展字段**：每条链接支持标题、简短描述、标签列表、关联批次号、收录日期等自定义字段，满足精细化检索与聚合统计需求。

**静态站点生成输出**：基于配置文件生成纯静态 HTML 导航页面，无需后端服务即可部署至任意 Web 服务器或 CDN，访问速度快且安全可控。

**全文检索引擎集成**：集成轻量级全文检索组件（支持中文分词），允许用户在当前收录范围内对标题、描述及标签执行关键词搜索。

**批量导出与备份**：支持将全部或指定批次的链接数据导出为 CSV、JSON 或 Markdown 表格格式，便于离线存档或迁移至其他工具。

## 应用场景

技术文档聚合站：技术团队可将日常积累的参考文档、API 手册、博客文章等外部链接按项目或技术栈分类入库，生成团队内部共享的知识导航页，替代浏览器书签的零散管理方式。

学术研究参考文献管理：研究人员在文献调研阶段收集的大量论文链接、数据网站、工具仓库可通过 WebLink Navigator 按研究主题批次录入，配合搜索功能快速定位特定领域资源，提高文献回顾效率。

开源项目生态展示：开源项目维护者可将依赖库、插件市场、社区论坛、示例项目等生态相关链接统一收录至项目的导航目录，帮助新贡献者快速了解项目周边工具链与学习路径。

运维监控仪表盘外链集成：运维人员可将常用的监控面板 URL、日志查询入口、报警配置文档集中管理，结合链接健康检测功能及时发现外部依赖服务的可用性变化。

## 快速开始

以下命令序列适用于 Linux/macOS 及 Windows WSL 环境，假定已安装 Git 与 Node.js 运行时。

```bash
# 克隆项目仓库
git clone https://github.com/weblink-navigator/weblink-navigator.git
cd weblink-navigator

# 安装项目依赖（使用 npm）
npm install

# 执行本地开发服务器，预览资源导航页面
npm run dev
```

执行上述三步后，打开浏览器访问 http://localhost:3000 即可查看当前批次的资源导航界面。生产环境构建命令为 `npm run build`，输出目录为 `./dist`，可将其内容部署至任何静态托管服务。

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | >= 18.12.0 LTS | 项目运行时与构建工具链的基础环境 |
| npm | >= 8.19.0 | 包管理器，用于安装项目依赖及执行脚本 |
| Git | >= 2.30.0 | 版本控制工具，用于克隆仓库及提交更新 |

| 网络访问 | 外网连通 | 用于初次下载依赖包及克隆仓库，运行时无需持续联网（除链接健康检测外） |

## 项目结构

weblink-navigator/
├── config/                              # 全局配置文件目录
│   ├── site.toml                        # 站点标题、描述、语言、主题颜色等基础配置
│   ├── categories.toml                  # 资源分类方案（技术/学术/运维/工具等）定义
│   └── batch-52.toml                    # 第52批次的专属配置（导入规则与标签映射）
├── data/                                # 资源数据存储目录
│   ├── batches/                         # 按批次存放原始导入文件
│   │   └── 52/                          # 第52批次原始数据及元信息
│   │       ├── manifest.json            # 批次号、导入时间、条目数、来源说明
│   │       └── raw-urls-52.txt          # 原始URL列表（保留大小写与扩展名）
│   ├── enriched/                        # 富化后的资源条目（含标题、描述、标签）
│   │   └── 52-enriched.json             # 第52批富化数据，数组结构每条包含url/title/desc/tags
│   └── cache/                           # 链接健康检测缓存（记录状态码与响应时间）
│       └── health-52.json               # 第52批链接最近一次检测结果快照
├── src/                                 # 源代码目录
│   ├── core/                            # 核心逻辑模块
│   │   ├── importer.js                  # 批次导入处理器，支持TXT/CSV/JSON格式
│   │   ├── enricher.js                  # 元数据富化引擎（调用外部API或本地规则）
│   │   ├── health-checker.js            # 链接可达性检测调度器，支持并发控制
│   │   └── search-index.js              # 全文索引构建与查询接口（基于分词库）
│   ├── generators/                      # 静态站点生成器
│   │   ├── page-builder.js              # HTML页面构建主流程
│   │   ├── home-view.js                 # 首页视图渲染（含分类统计与最新批次）
│   │   ├── detail-view.js               # 单条资源详情页模板
│   │   └── sitemap-generator.js         # 生成sitemap.xml供搜索引擎爬取
│   ├── templates/                       # 前端模板与样式文件
│   │   ├── layouts/                     # 布局骨架（头部、导航、底部）
│   │   ├── components/                  # 可复用UI组件（卡片、标签、搜索框）
│   │   └── assets/                      # 静态资源（CSS样式表、JavaScript交互脚本）
│   └── utils/                           # 通用工具函数
│       ├── url-helper.js                # URL规范化、域名提取、路径解析
│       ├── date-helper.js               # 时间格式化与时区转换

## 常见问题

**Q：如何将已有浏览器书签批量导入项目？**

A：项目未直接提供浏览器书签解析器，但您可以将书签导出为 HTML 文件，然后使用 Python 或 Node.js 脚本从中提取 URL 列表，整理为纯文本或 CSV 格式后放入 `data/batches/<批次号>/` 目录，再执行 `npm run import -- --batch <批次号>` 命令完成导入。后续版本计划增加书签 HTML 的直接解析支持。

**Q：链接健康检测显示大量超时或拒绝连接，是否表示链接已彻底失效？**

A：检测结果受网络环境、目标站点防火墙策略及检测并发数影响。单次超时不一定代表链接永久失效，建议在非高峰时段重新运行检测（`npm run health-check -- --retry`），并适当调整 `config/site.toml` 中的超时阈值与重试次数。对于持续失败的链接，人工访问确认后再决定保留或移除。

**Q：能否在静态生成后仍保留搜索功能？**

A：项目生成的静态页面中搜索功能通过客户端 JavaScript 实现——构建时将全部资源数据嵌入为 JSON 索引文件，页面加载后由前端脚本执行本地搜索。该方案无需后端服务，但受限于索引文件大小（建议单批次不超过 2000 条）。对于更大的数据集，可考虑额外部署后端搜索服务或对接第三方搜索引擎。
