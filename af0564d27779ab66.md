# WebIndex Central

WebIndex Central 是一个面向技术研究者、信息分析人员和数据归档工作者的高质量外链资源汇总与导航工具。该项目不生产内容，而是通过人工筛选与自动化校验相结合的方式，对散布于互联网各处的深度技术文章、案例分析及数据报告进行结构化整理，解决技术信息碎片化、优质内容难以追溯的痛点。

该项目定位于中长尾技术资源的发现与重用。目标用户包括需要快速建立领域知识地图的架构师、进行竞品技术栈分析的产品经理、以及需要为文档或报告寻找权威引用源的技术撰稿人。通过将异构的 URL 资源纳入统一的元数据管理框架，WebIndex Central 使原本孤立的信息节点形成可查询、可关联的知识网络。

## 功能概览

**批量资源注入**：支持通过命令行或 API 批量导入 URL 列表，系统自动完成去重、存活检测与初步分类。

**智能分类标记**：基于 URL 路径特征与预设规则库，为每个资源自动添加技术领域、内容类型（如案例分析、API 文档、性能调优）等标签。

**全文元数据提取**：对每个目标页面进行可配置的深度抓取，提取标题、发布时间、正文摘要及关键代码块引用。

**自定义视图构建**：允许用户基于标签、时间范围或来源域名创建动态集合，并将集合生成为静态 HTML 仪表板或 JSON 馈送。

**校验与健康监控**：定期回访已收录资源，检测链接可用性及内容变更，生成变更报告并通过 Webhook 通知订阅者。

**只读镜像缓存**：为高频访问的资源提供只读的文本缓存层，降低原始服务器的请求压力，同时确保内容在源站临时不可用时仍可被查阅。

**开放查询接口**：提供基于 RESTful 风格的查询 API，支持按关键字、标签组合及时间范围进行精确检索。

## 应用场景

**技术选型调研**：当团队需要评估多个中间件或数据库方案时，可通过 WebIndex Central 快速聚合来自技术博客、官方文档和社区评测的相关文章，形成对比阅读列表，显著缩短信息收集周期。

**遗留系统文档重建**：对于缺乏维护的遗留项目，工程师可以利用本工具追溯散落在不同平台的设计说明和问题排查记录，为重构决策提供历史依据。

**安全漏洞影响面分析**：安全响应团队在收到 CVE 通报后，可通过检索特定组件名称或版本号，快速定位内部知识库中所有涉及该组件的资源链接，评估受影响的业务范围。

**技术资讯周报自动生成**：运营人员可配置定时任务，自动拉取过去七天内新增的资源列表，依据标签分组后生成带摘要的周报草稿，减少人工编排工作量。

## 快速开始

以下步骤将引导您在本地环境中启动 WebIndex Central 的基础实例。

```bash
# 克隆项目仓库至本地
git clone https://github.com/webindex-central/webindex-core.git
cd webindex-core

# 安装项目依赖（使用 pip 与 requirements.txt）
pip install -r requirements.txt

# 以开发模式运行内置的测试服务器
python run.py --mode dev --port 8080
```

执行完上述命令后，访问控制台输出的本地地址即可进入仪表板界面。默认情况下，系统会加载一组示例资源用于展示功能。

## 安装要求

生产环境部署前，请确保基础环境满足以下依赖条件。开发环境可使用内置的 SQLite 后端，无需额外安装数据库服务。

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行时环境，建议使用 pyenv 管理版本 |
| PostgreSQL | 14.x 及以上 | 生产环境推荐使用的联机事务处理数据库 |
| Redis | 7.0 及以上 | 用于缓存查询结果和存储临时任务队列 |
| Celery | 5.3.x | 分布式任务队列框架，处理异步抓取与校验任务 |
| BeautifulSoup4 | 4.12.x | HTML 内容解析库，用于提取文章正文与元数据 |
| Requests | 2.31.x | HTTP 客户端库，处理所有出站网络请求 |
| Pytest | 7.4.x | 单元测试与集成测试框架（仅开发环境需要） |

## 项目结构

项目采用分层架构设计，核心模块与辅助工具分离，便于独立演进和测试。

```
webindex-core/
├── run.py                          # 项目启动入口，包含开发与生产模式切换
├── requirements.txt                # Python 依赖清单，锁定所有第三方库版本
├── config/                         # 配置管理模块
│   ├── default.py                  # 默认配置项（日志级别、抓取超时、分页大小）
│   ├── production.py               # 生产环境覆盖配置（密钥、数据库连接池）
│   └── development.py              # 开发环境覆盖配置（启用调试工具、热重载）
├── core/                           # 核心业务逻辑层
│   ├── fetcher/                    # 抓取子模块：负责 HTTP 请求与重试策略
│   │   ├── client.py               # 封装 requests.Session，支持代理与限速
│   │   └── parser.py               # 基于 BeautifulSoup4 的内容解析器
│   ├── indexer/                    # 索引子模块：管理资源元数据的入库与更新
│   │   ├── models.py               # SQLAlchemy 数据表定义（Resource, Tag, Snapshot）
│   │   └── crud.py                 # 增删改查操作的事务性封装
│   └── scheduler/                  # 调度子模块：协调周期性任务（健康检查、报告生成）
│       ├── tasks.py                # Celery 任务声明（fetch_task, check_task）
│       └── beat_schedule.py        # Celery Beat 定时表达式配置
├── api/                            # 对外接口层
│   ├── routes/                     # 路由定义（按资源类型拆分）
│   │   ├── resources.py            # /api/v1/resources 端点实现
│   │   └── collections.py          # /api/v1/collections 端点实现
│   └── schemas/                    # Pydantic 请求与响应模型校验
│       └── resource_schema.py      # 输入输出的字段约束与序列化规则
├── utils/                          # 通用工具函数集
│   ├── logger.py                   # 统一日志格式化与轮转策略
│   ├── validators.py               # URL 规范化与域名黑名单校验
│   └── text_processor.py           # 摘要生成与关键词提取辅助函数
├── tests/                          # 测试套件（与源码目录保持平行结构）
│   ├── unit/                       # 单元测试（针对单个函数或类）
│   └── integration/                # 集成测试（涉及数据库与网络请求）
├── docs/                           # 文档源文件（Markdown 格式）
│   ├── getting_started.md
│   ├── usage/
│   └── contributing/
└── scripts/                        # 运维辅助脚本
    ├── init_db.py                  # 初始化数据库表结构与默认数据
    └── import_batch.py             # 批量导入外部 URL 列表的命令行工具
```

## 常见问题

**如何自定义抓取请求的 User-Agent 和超时时间？**

您可以在 config/development.py 或 config/production.py 中覆盖 FETCHER_USER_AGENT 和 FETCHER_TIMEOUT 变量。默认 User-Agent 为 WebIndexCentral/1.0 (+https://webindex.example.com)，超时时间为 15 秒。修改后需要重启服务进程。

**系统如何处理已失效的链接？**

健康监控任务会以可配置的周期（默认每 7 天）对所有资源发起 HEAD 请求。当连续两次检查均返回 4xx 或 5xx 状态码时，系统会将该资源标记为「unreachable」，并在仪表板中以灰显样式提示。您可以通过 API 的 force_refresh 参数手动触发即时校验。

**能否将数据迁移至其他关系型数据库？**

核心数据模型基于 SQLAlchemy ORM 构建，理论上支持所有主流关系型数据库（MySQL、SQLite、Oracle 等）。您只需在配置中修改 SQLALCHEMY_DATABASE_URI 连接串，并安装对应的数据库驱动（如 psycopg2-binary 或 pymysql）。建议在切换数据库后先运行迁移脚本以验证表结构兼容性。
