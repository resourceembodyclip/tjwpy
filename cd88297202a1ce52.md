# TechNav Resource Aggregator

TechNav is a curated technical resource navigation system designed for developers, technical researchers, and IT professionals who need efficient access to high-quality technical articles, tutorials, and documentation. The project addresses the common problem of fragmented technical information scattered across multiple platforms by providing a unified, searchable index of curated technical content.

This repository serves as a comprehensive resource hub containing over 180 categorized technical article links, with each entry manually reviewed and annotated for relevance, technical depth, and practical utility. The project emphasizes mobile-first accessibility, making it suitable for on-the-go reference during development work or technical research.

## 功能概览

**分类索引系统** - Articles are organized into logical categories including programming languages, framework tutorials, system architecture, database optimization, and DevOps practices, enabling rapid discovery of relevant content.

**移动端优化阅读** - All resources are sourced from a mobile-optimized platform ensuring consistent reading experiences across smartphones, tablets, and desktop devices.

**全文检索支持** - Integrated search functionality allows users to filter the 180+ resource collection by keywords, technical tags, and content types.

**定期更新机制** - The resource index receives weekly updates with new technical articles, ensuring the content remains current with evolving technology trends.

**标签过滤系统** - Each article is tagged with multiple technical labels such as language, framework, difficulty level, and topic area for precise filtering.

**收藏与书签功能** - Users can mark favorite articles and create custom reading lists within the navigation interface.

**共享与协作** - The resource index supports team sharing, enabling development groups to maintain a collective knowledge base.

**内容摘要预览** - Each listing includes a concise abstract, allowing users to assess article relevance before navigating to the full content.

## 应用场景

**日常技术学习** - Developers can use TechNav as their primary learning resource, accessing tutorials and technical deep-dives across multiple programming domains during skill development sessions.

**项目技术选型参考** - When evaluating frameworks, libraries, or architectural approaches, teams can reference curated articles comparing technologies, reading implementation experiences, and reviewing performance benchmarks.

**故障排查辅助** - System administrators and SRE engineers can quickly locate troubleshooting guides and diagnostic articles when responding to production incidents or resolving complex technical issues.

**团队知识库建设** - Technical leads can curate article collections relevant to their team's technology stack, creating an internal knowledge repository for onboarding new members and maintaining coding standards.

**技术会议材料准备** - Speakers and technical trainers can gather reference materials, code examples, and case studies from the curated collection when preparing presentations or workshop content.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/technav-resources/technav-aggregator.git

# Navigate to project directory
cd technav-aggregator

# Install dependencies
npm install

# Start the development server
npm run dev

# Alternatively, for production build
npm run build
npm start
```

## 安装要求

| 依赖 | 必需版本 | 说明 |
|------|----------|------|
| Node.js | >= 18.0.0 | JavaScript 运行时环境，用于构建和运行导航系统 |

| Git | >= 2.30 | 版本控制系统，用于代码管理 |

## 贡献指南

## 常见问题

**Q: 如何确保所有收录的链接保持有效和内容更新？**

A: 系统内置了自动化链接检查器，每 72 小时对所有收录 URL 进行可达性检测。对于返回 404 或 5xx 状态码的链接，会自动标记为失效并移出主索引。同时通过 RSS 订阅和定期人工审核确保内容的新鲜度，发现内容更新时会自动同步最新的摘要信息。

**Q: 搜索功能的原理是什么？支持的搜索语法有哪些？**

A: 搜索基于 Elasticsearch 全文检索引擎，支持中文分词和英文词干提取。搜索语法包括：使用引号进行精确短语匹配（例如 "microservices architecture"），使用减号排除关键词（例如 java -spring），使用星号通配符（例如 react*），以及使用 AND/OR 布尔运算符组合多个关键词。搜索结果按相关度、发布时间和收藏热度综合排序。

**Q: 如何将 TechNav 部署到企业内部网络供团队使用？**

A: 提供了完整的 Docker 容器化部署方案。只需确保内网环境满足安装要求中的依赖版本，配置好 .env 文件中的数据库连接字符串和 Elasticsearch 端点，执行 docker-compose up -d 即可启动全部服务。支持通过环境变量覆盖所有配置项，便于集成到现有的 CI/CD 流水线中。详细步骤参考 docs/deployment/ 目录下的企业内网部署指南。
