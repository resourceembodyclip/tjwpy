# WebFront Knowledge Aggregator

WebFront Knowledge Aggregator 是一个面向全栈开发者和技术研究人员的综合性技术资源导航与外链汇总系统。该项目定位于对分散在互联网各处的优质技术文章、教程、案例分析和开发文档进行系统性收录与分类管理，帮助开发者在碎片化信息环境中快速定位高质量学习材料。

本项目面向的典型用户包括前端与后端工程师、运维人员、技术架构师以及计算机专业学生。项目本身不生产原创内容，而是通过人工筛选与自动化采集相结合的方式，将来自多个数据源的技术文档按照统一格式进行归集，并提供清晰的检索路径。当前批次为第 25/56 批，累计收录资源链接 180 条，覆盖移动端开发、后端架构、数据库调优、DevOps 实践等多个技术领域。

## 功能概览

**按技术领域分类浏览** 系统将收录的资源按照前端、后端、数据库、运维、架构设计等维度进行标签化分类，用户可根据自身技术栈快速筛选相关内容。

**全文元数据检索** 针对每一条收录资源，系统提取标题、发布时间、来源域名、内容摘要等元数据字段，支持基于关键词的全文检索，帮助用户在 180 条链接中精准定位目标文章。

**批次管理与版本追踪** 每批资源导入均记录批次号与导入时间戳，用户可查看当前批次（25/56）的完整资源清单，并追踪历史批次的收录情况。

**外链健康度检测** 系统每日定时对已收录的 URL 进行可达性探测，自动标记响应超时或返回异常状态码的链接，并在管理面板中高亮显示。

**响应式资源展示面板** 前端展示层基于移动优先原则设计，适配桌面端与移动端浏览器，资源列表支持按标题、来源、日期排序，并提供简明摘要预览。

**数据导出与订阅** 支持将当前批次的资源列表导出为 CSV 或 JSON 格式，便于用户离线查阅或二次加工；同时提供 RSS 订阅接口，用户可实时获取新增资源推送。

## 应用场景

技术团队内部知识库建设：团队技术负责人可将本项目部署为内部知识导航站点，统一收录团队推荐的对外技术文章与官方文档链接，替代零散的浏览器书签，提升团队知识沉淀的规范性与可访问性。

个人开发者学习路径管理：全栈开发者在日常技术阅读中积累大量待读或已读的技术资料，通过本项目提供的批次管理与分类浏览功能，可系统性地整理个人学习资源库，并追踪每篇文章的阅读完成状态。

技术社区内容聚合展示：技术博客或开发者社区运营方可利用本项目作为内容聚合的后台引擎，将投稿或爬取获得的外链统一格式化输出，降低内容管理成本，同时为访客提供结构化的外链导航服务。

自动化资源监控与告警：运维或安全团队可将本项目集成至监控体系中，利用链接健康度检测功能定期巡检依赖的外部文档链接是否可用，及时发现失效外链并触发告警，避免对外文档引用断裂。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，要求系统已安装 Git、Node.js 18.x 及以上版本与 npm 或 yarn 包管理器。

```bash
# 克隆项目仓库至本地
git clone https://github.com/webfront/knowledge-aggregator.git
cd knowledge-aggregator

# 安装项目依赖（使用 npm）
npm install

# 复制环境变量模板并填充必要配置
cp .env.example .env

# 执行数据库初始化和资源索引构建
npm run setup

# 启动开发服务器（默认监听端口 3000）
npm run dev
```

生产环境部署请使用 `npm run build` 构建静态资源，并通过 `npm start` 启动生产服务。

## 项目结构

```
knowledge-aggregator/
├── apps/
│   ├── web/                               # 前端展示应用（Next.js）
│   │   ├── pages/                         # 页面路由：首页、分类页、详情页、关于页
│   │   ├── components/                    # 可复用 UI 组件：资源卡片、搜索栏、分页器
│   │   └── styles/                       # 全局样式与主题变量
│   └── api/                               # 后端服务应用（Express.js）
│       ├── routes/                        # RESTful 路由定义：资源、批次、健康检查
│       ├── controllers/                   # 请求处理控制器：数据校验、业务逻辑编排
│       └── services/                      # 服务层：数据库访问、外链探测、缓存管理
├── packages/
│   ├── shared-types/                      # 前后端共享 TypeScript 类型定义
│   ├── data-import/                       # 批量资源导入工具（支持 CSV / JSON）
│   └── link-checker/                      # 外链健康度检测独立模块（定时任务）
├── scripts/
│   ├── setup-db.sql                       # 数据库初始化脚本（建表、索引、种子数据）
│   └── migrate-batch.js                   # 批次迁移与数据校验脚本
├── config/
│   ├── default.json                       # 默认配置：端口、日志级别、分页大小
│   └── production.json                    # 生产环境覆盖配置
├── docs/                                  # 完整文档（见文档导航章节）
├── tests/                                 # 单元测试与集成测试（Jest + Supertest）
├── .env.example                           # 环境变量示例文件
├── docker-compose.yml                     # 本地开发环境编排（PostgreSQL + Redis）
├── Dockerfile                             # 生产镜像构建定义
├── package.json                           # 项目依赖清单与脚本定义
├── tsconfig.json                          # TypeScript 编译配置
└── README.md                              # 本文档
```

## 常见问题

**问：导入资源时提示 URL 格式校验失败，但链接在浏览器中可以正常访问，是什么原因？**

答：系统对 URL 格式校验采用严格的正则表达式规则，要求协议头（http:// 或 https://）必须小写，且域名部分不允许包含大写字符。请检查您提交的 URL 是否包含大写字母或多余的空格。若确认链接本身无误，可暂时将校验规则调整为宽松模式，在配置文件中将 `strictUrlValidation` 设置为 `false` 后重试。

**问：外链健康度检测任务执行时，大量链接返回超时，但手动访问是正常的，如何解决？**

答：这通常是由于检测任务并发数过高导致目标服务器限流或触发反爬机制。请调整 `config/default.json` 中的 `linkChecker.concurrency` 参数，将并发数从默认的 10 降低至 3 或 5，同时增加 `linkChecker.timeout` 超时阈值至 10000 毫秒。若检测目标均为同一域名，建议在 `linkChecker.rateLimit` 中配置请求间隔（如 200 毫秒）以避免被目标服务器封禁。

**问：搜索功能无法检索到新导入的资源，但资源列表页面中已正常显示，如何修复索引？**

答：系统默认使用 PostgreSQL 的全文检索索引，资源导入后索引更新存在约 5 分钟的异步延迟。若需立即生效，可在管理后台点击「重建索引」按钮，或执行 `npm run rebuild-search-index` 命令手动触发全量索引重建。若问题持续存在，请检查数据库连接池是否健康，以及 `searchVector` 字段在导入时是否正确生成。
