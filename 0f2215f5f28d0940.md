# LinkMaster Pro

LinkMaster Pro 是一个面向技术团队与内容运营者的外链资源聚合与管理平台，专注于将散落在各类技术文档、博客、内部知识库中的外部优质链接进行统一采集、分类归档与健康度监控。该项目并非一个简单的书签管理器，而是一个具备链接有效性检测、元数据自动抓取、标签体系与团队共享能力的开源外链中台。

项目定位为技术资源导航站与内部知识库的链接网关，解决的是中大型研发组织中文档散乱、外链失效无法感知、新成员难以快速找到权威参考资源等实际问题。LinkMaster Pro 既可作为独立站点运行，也可通过 REST API 嵌入现有 Confluence、语雀或自建文档系统，帮助团队将被动查阅转变为主动发现。

## 功能概览

**批量链接导入与解析**：支持从 Markdown、CSV、JSON 及浏览器书签 HTML 文件中批量导入链接，自动解析页面标题、摘要描述、语言类型及发布日期，减少人工录入成本。

**链接健康度实时监控**：内置分布式定时任务调度器，按可配置周期（小时/天/周）对全量链接发起 HEAD/GET 请求，检测 HTTP 状态码、响应时间与页面重定向链，异常时触发告警通知。

**多维度标签与分类体系**：提供无限层级的分类目录与自由标签双模管理，支持正则表达式自动打标，便于按技术领域、团队归属、文档状态等维度快速筛选。

**全文检索与高级过滤器**：基于倒排索引实现链接标题、描述、标签及正文前 512 个字符的全文搜索，同时支持按状态码、文件类型、更新时间、来源域名等十余种字段组合过滤。

**团队协作与权限控制**：基于 RBAC 模型设计，支持项目组、部门、个人三级资源隔离，可赋予成员只读、编辑、审核或管理员权限，操作日志全量留存。

**开放 API 与 Webhook 集成**：提供 Swagger 规范的标准 RESTful API，覆盖链接增删改查、批量操作与状态查询；支持配置 Webhook 将链接变更事件推送至飞书、钉钉、Slack 或企业微信。

**数据可视化看板**：内置管理仪表盘，展示链接总数、健康率、响应耗时分布、热门标签及新增趋势图，帮助团队快速了解资源库整体状况。

## 应用场景

**技术文档库外链治理**：当团队维护超过百篇的技术文档或 API 手册时，其中引用的外部参考链接极易因目标站点改版、域名过期而失效。LinkMaster Pro 可定期扫描文档源文件中的全部链接，自动标记失效项并生成报告，运维人员可在文档更新前主动修复。

**新人入职学习路线导航**：将公司内部推荐的编程规范、框架官网、最佳实践文章、视频教程等链接统一录入系统，并按照前端、后端、运维、算法等岗位分类。新人入职后通过标签筛选即可获得完整学习路径，避免在杂乱的书签中迷失。

**开源项目外部依赖溯源**：对于依赖大量第三方库、工具链与参考实现的开源项目，维护者可将所有依赖项的官网、文档、Issue 追踪地址及镜像站收录至 LinkMaster Pro，当某个依赖仓库迁移或下线时，系统第一时间通知所有相关维护者。

**技术周报与资讯聚合**：运营者或技术负责人可每日将阅读到的优质技术博文、Release Note、安全通告等链接快速入库，团队订阅相应标签后通过 RSS 或邮件摘要方式获取每周精选，减少信息过载。

## 快速开始

以下步骤指导您在本地开发环境快速启动 LinkMaster Pro 服务。

```bash
# 克隆项目仓库
git clone https://github.com/linkmaster-pro/linkmaster-pro.git

# 进入项目目录
cd linkmaster-pro

# 安装依赖（使用 pip 与 requirements.txt）
pip install -r requirements.txt

# 初始化配置文件和环境变量
cp .env.example .env
cp config/application.example.yml config/application.yml

# 执行数据库迁移脚本
python manage.py migrate

# 导入默认标签和分类种子数据
python manage.py seed --init

# 启动开发服务器（默认监听 8000 端口）
python manage.py runserver 0.0.0.0:8000
```

访问 http://localhost:8000 即可进入管理控制台，默认管理员账号 admin 密码请从 .env 文件中获取初始值，首次登录强制修改。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.10 ~ 3.12 | 核心运行环境，使用 asyncio 异步 I/O 处理网络请求 |
| PostgreSQL | 15.x 及以上 | 主数据库，存储链接元数据、标签关系及用户信息 |
| Redis | 7.x 及以上 | 缓存会话、分布式锁及定时任务状态存储 |
| Elasticsearch | 8.x 及以上 | 全文检索引擎，可选组件，未安装时降级为 SQL 模糊匹配 |
| Celery Broker (RabbitMQ/Redis) | RabbitMQ 3.12 或 Redis 7.x | 异步任务队列，用于链接健康度检测与抓取 |
| Nginx | 1.24 及以上 | 生产环境反向代理与静态资源服务，开发环境可忽略 |
| Docker / Docker Compose | 24.x 及以上 | 容器化部署方案，生产环境推荐使用 |
| Git | 2.30 及以上 | 版本控制及自动化部署拉取 |
| Supervisor | 4.2 及以上 | 进程守护，用于生产环境管理 Gunicorn 与 Celery Worker |

## 常见问题

**Q：LinkMaster Pro 能否处理需要登录或带有反爬机制的链接？**

A：健康度检测模块默认仅进行轻量级 HEAD 请求或带超时限制的 GET 请求，不执行 JavaScript 渲染。对于需要鉴权的内部链接，您可以在配置文件中为特定域名设置自定义请求头（如 Cookie 或 Token），但请注意不要将敏感凭证提交至版本控制。若目标站点有严格反爬策略，建议在配置中降低检测频率或将该域名加入白名单跳过检测。

**Q：导入大量链接时系统性能会受影响吗？**

A：批量导入接口采用异步任务队列设计，用户提交导入请求后立即返回任务 ID，实际解析与入库操作由 Celery Worker 在后台处理。单次导入支持最高 10 万条记录，生产环境建议结合 Redis 限流，避免瞬时压力过高。同时，Elasticsearch 索引采用批量写入优化，确保检索响应时间维持在 200 毫秒以内。

**Q：如何从现有浏览器书签或 Notion 数据库迁移数据？**

A：项目提供独立的迁移脚本 `scripts/migrate_legacy.py`，支持读取 Chrome/Firefox 导出的 HTML 书签文件，以及通过 Notion API 拉取数据库属性。您需要根据源格式准备对应的映射配置文件，将书签文件夹名称映射为标签，标题映射为链接名称，URL 和描述字段自动对应。迁移完成后，系统会输出统计报告，方便核对总数与遗漏项。
