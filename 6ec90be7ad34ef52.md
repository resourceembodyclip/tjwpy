# LinkApex 技术资源导航系统

LinkApex 是一个面向开发者和技术研究人员的结构化外链资源聚合平台，专注于对分散于互联网各处的技术文档、教程文章、案例解析和代码示例进行系统性归集与索引。该项目不生产原创内容，而是通过人工筛选与自动化校验相结合的方式，将高质量外部技术资源按照主题域、内容类型和质量等级进行分层组织，最终以清晰可检索的目录树形态呈现给终端用户。

项目定位为技术资源的外链汇总与导航中间件，主要服务于以下三类用户：需要进行技术选型调研的架构师、希望快速查阅特定技术点最佳实践的研发工程师，以及正在学习某一技术栈并需要大量阅读材料的初学者。LinkApex 通过统一的条目描述模板、标签体系和相关性评分，帮助用户在海量链接中定位到真正有价值的阅读对象，显著降低信息筛选的时间成本。

## 功能概览

**多级分类目录体系**：所有收录链接按技术领域、应用场景和难度等级三个维度进行归类，支持用户通过目录树逐级下钻，亦可通过全局搜索直接定位目标分类节点。

**资源元数据标注**：每个外链条目均附带来源域名、内容摘要、发表日期、技术栈标签和阅读时长预估，便于用户在不离开导航页面的前提下评估链接的参考价值。

**每日自动校验**：系统后台通过定时任务对所有已收录链接进行可达性检测和响应状态码校验，自动标记失效链接并生成报告，确保资源列表的鲜活度和可用性。

**用户自定义收藏夹**：注册用户可将常用资源链接加入个人收藏夹，并支持添加自定义备注标签，实现个性化资源管理。

**批量导入与去重**：支持通过 CSV 或 JSON 格式批量导入外部链接清单，系统自动识别并合并重复条目，避免冗余数据污染索引库。

**访问热力统计**：每个外链条目独立记录点击次数和最后访问时间，以热力图形式展示高频访问资源，辅助识别社区关注热点。

**内容快照预览**：对部分稳定性较高的技术文档类链接，系统提供静态 HTML 快照存储功能，在源站不可访问时仍可阅读正文内容。

**开放 API 接口**：提供 RESTful API 供第三方开发者查询和检索资源目录，支持按分类、标签、关键词进行组合过滤，便于集成至其他工具链。

## 应用场景

技术选型调研阶段，架构师需要横向对比多个开源方案的功能特性和社区活跃度。LinkApex 的"方案对比"分类下汇集了大量第三方评测文章和官方文档入口，用户可在同一目录层级内快速获取多个备选方案的参考资料，显著提升调研效率。

日常开发过程中，工程师遇到特定功能实现难题时，往往需要在搜索引擎中反复尝试不同关键词才能找到可用的代码示例或解决思路。LinkApex 的标签检索系统允许用户通过"语言+框架+功能"的组合标签直接命中经过筛选的高质量解答链接，减少无效点击。

技术团队内部知识库建设场景中，团队负责人可以利用 LinkApex 的批量导入和自定义分类功能，将团队长期积累的收藏夹、书签和文档引用统一迁移至平台内，形成团队共享的外部参考资料索引，新成员入职时可快速了解团队推荐的技术阅读清单。

## 快速开始

以下指令适用于 Linux 与 macOS 环境，Windows 用户建议使用 WSL2 或 Git Bash 执行。

```bash
git clone https://github.com/linkapex/linkapex.git
cd linkapex
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver 0.0.0.0:8000
```

执行上述命令后，服务将在本地 8000 端口启动。打开浏览器访问 http://localhost:8000 即可进入资源导航首页。首次启动时系统会自动创建默认分类目录并导入示例资源数据。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，低于 3.9 版本将无法解析类型注解语法 |
| PostgreSQL | 12.x 及以上 | 主数据库，用于存储资源条目、用户数据和访问统计 |
| Redis | 6.0 及以上 | 缓存会话数据与热力统计计数，非必需但强烈建议启用 |
| Celery Worker | 5.2 及以上 | 异步任务队列，用于执行每日链接校验和快照生成 |
| Nginx | 1.18 及以上 | 生产环境反向代理与静态资源服务，开发环境可跳过 |
| Node.js | 16.x 及以上 | 仅用于前端资源构建，后端开发可忽略此项 |
| Docker | 20.10 及以上 | 容器化部署方案依赖，非必需但便于环境一致性管理 |

## 常见问题

启动服务后访问页面提示数据库连接失败应如何处理

检查 PostgreSQL 服务是否正常运行，使用 systemctl status postgresql 或 brew services list 查看状态。确认 settings.py 中的数据库名称、用户名、密码和主机地址与实际配置一致。若使用 Docker 部署，请确保容器已正确暴露 5432 端口且卷挂载路径可写。

每日链接校验任务报告大量 404 状态码，是否需要人工介入

校验任务仅做状态记录和标记，不会自动删除条目。收到报告后建议先手动访问部分链接确认是否为临时性故障，对于确已永久失效的链接可在管理后台将其状态标记为"已失效"并移入待审核队列。项目维护者会定期清理长期失效条目，也欢迎社区提交替代链接进行替换。

如何将现有浏览器书签批量导入至 LinkApex

目前支持通过管理后台的 CSV 导入功能完成批量操作。请将书签导出为 CSV 文件，每行包含标题、URL、分类路径和标签列表四列，然后使用 scripts/import_links.py 脚本执行导入，或通过后台界面逐批上传。分类路径若不存在系统会自动创建。
