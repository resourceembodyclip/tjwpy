# WebLink Navigator

WebLink Navigator 是一个面向技术研究者、开发者和内容策展人的结构化外链资源聚合系统。该项目通过对海量技术文章、行业动态与工程实践类 URL 进行规范化采集、分类标注与索引构建，为特定垂直领域提供高密度、可检索的外部知识库入口。其核心定位并非爬虫或全文存储工具，而是一种轻量级、人工校验与自动化标记相结合的资源导航中间件，适用于需要持续追踪特定域名下内容更新节奏的工作流。

本项目专为以下场景设计：当用户面对大量来自同一内容源的文章链接时，WebLink Navigator 能够提供基于 URL 模式解析、批次管理和元数据快照的辅助能力，帮助团队快速建立外部参考资料的时间轴与主题聚类视图。项目自身不依赖外部数据库，所有索引信息存储于本地结构化文件中，便于版本控制与协作分发。

## 功能概览

批次化资源收录：支持按批次编号（如第 51/56 批）对原始 URL 列表进行整体导入，自动记录收录时间戳与来源批次属性，便于多轮资源整合过程中的溯源与去重。

URL 模式解析引擎：自动识别文章 ID 与扩展名模式，对形如 /Article/details/{id}.sHtML 的路径进行正则提取，生成纯数字标识符作为资源主键，为后续分类提供稳定索引基础。

轻量级元数据注入：允许用户为每个链接补充自定义标签、优先级标记和阅读状态，所有元数据以 YAML front matter 形式存储于对应的索引记录中，不侵入原始资源。

静态站点生成适配：内置模板渲染器，可将索引数据输出为 HTML 目录页或 Markdown 清单文件，适用于生成项目内部知识库主页或团队周报中的引用附录。

多维度筛选视图：基于文章 ID 数值区间、收录批次、自定义标签等维度提供过滤接口，支持命令行与 HTTP 查询两种模式，便于集成到现有监控或通知系统。

原始 URL 保真输出：在资源导出、备份或迁移过程中，严格保持用户输入 URL 的字符串原貌，不进行协议补全、大小写转换或路径规范化，确保外部引用链路的确定性。

完整性校验工具：提供对已收录 URL 列表的断链检测与状态码检查功能，生成可用性报告，帮助维护资源库的健康度。

## 应用场景

技术团队外部知识库构建：研发团队在调研某一新技术领域时，往往会积累大量来自行业媒体或技术博客的文章链接。WebLink Navigator 可作为该流程的收口工具，将分散在文档、即时通讯或邮件中的 URL 统一纳入批次管理，并生成可共享的 Markdown 索引文件，方便新人快速了解领域关键资料。

内容运营与编辑工作流：内容编辑在策划专题或撰写行业分析报告时，需要引用大量外部案例。使用本项目的批次导入功能，可一次性将数十篇参考文章录入系统，通过自定义标签区分“已引用”“待核实”“高价值”等状态，提高内容生产阶段的资料组织效率。

个人知识管理辅助：对于长期跟踪特定网站内容输出的研究者，本项目提供了一种基于 URL 模式的自动化归类方案。用户只需将新收集到的链接按批次提交，系统即可自动提取文章 ID 并生成时间排序视图，帮助研究者建立个人的外部文献年鉴。

开源项目文档外链治理：开源项目的 README 或 Wiki 中常包含大量外部参考链接。随着项目迭代，这些链接可能出现失效或内容迁移。WebLink Navigator 的完整性校验工具可定期扫描项目文档中的外链，输出存活状态报告，辅助维护者及时更新文档。

## 快速开始

以下指令适用于 Linux/macOS 及 Windows WSL 环境。请确保已安装 Git 与 Node.js（版本 16.0 以上）。

```bash
# 克隆项目仓库至本地
git clone https://github.com/your-organization/weblink-navigator.git

# 进入项目根目录并安装依赖
cd weblink-navigator
npm install

# 运行初始化向导，创建示例索引库并导入第一批资源
npm run init
npm run import -- --batch=51 --source=./samples/url-list-51.txt
```

执行上述命令后，系统将在 ./data 目录下生成以批次号命名的索引文件，并输出本次导入的统计摘要。如需启动本地 Web 预览界面，可运行 `npm run serve`，默认监听 127.0.0.1:3000。

## 项目结构

```
weblink-navigator/
├── bin/                                    # CLI 可执行入口
│   ├── cli.js                              # 主命令调度器，注册 import、export、check 子命令
│   └── commands/                           # 各子命令实现模块
│       ├── import.js                       # 批次导入逻辑，包含 URL 解析与 ID 提取
│       ├── export.js                       # 索引导出为 Markdown/JSON/HTML
│       └── check.js                        # 断链检测与状态报告生成器
├── lib/                                    # 核心库代码
│   ├── indexer/                            # 索引构建引擎
│   │   ├── batch.js                        # 批次对象模型，管理批次编号与资源列表
│   │   ├── parser.js                       # URL 模式解析器，支持正则提取与校验
│   │   └── serializer.js                   # 索引序列化与反序列化（YAML/JSON）
│   ├── storage/                            # 存储适配层
│   │   ├── file-system.js                  # 基于 Node.js fs 模块的读写实现
│   │   └── memory.js                       # 内存缓存，用于批量操作时的临时存储
│   ├── validator/                          # 校验工具集
│   │   ├── url-validator.js                # URL 格式规范校验
│   │   └── link-checker.js                 # HTTP 状态码检查，支持并发与超时控制
│   └── renderer/                           # 输出渲染器
│       ├── markdown.js                     # 将索引渲染为 Markdown 列表
│       └── html.js                         # 生成简易 HTML 导航页
├── data/                                   # 数据目录（自动生成）
│   ├── batches/                            # 按批次存储的索引文件
│   │   ├── batch-51.yaml                   # 第 51 批资源索引
```

## 常见问题

Q: 导入 URL 时提示 “URL 格式校验失败”，但链接在浏览器中可正常访问，如何解决？

A: 本项目在校验阶段会对 URL 进行基础格式检查，包括协议头（http/https）是否存在、域名是否包含非法字符等。您提供的这批链接均以 http:// 开头，符合规范。若出现校验失败，请检查链接是否包含未转义的空格或换行符。您也可通过 `--strict=false` 参数关闭严格校验模式，但建议在关闭前人工确认链接有效性。

Q: 如何将不同批次的资源合并导出为一个统一的 Markdown 文件？

A: 使用 `npm run export -- --merge --format=markdown --output=combined.md` 命令。该命令会扫描 ./data/batches/ 下所有批次文件，按文章 ID 数值升序合并，并去除重复条目。最终生成的 Markdown 文件可直接用于项目文档或团队周报。

Q: 项目是否支持 Windows 原生命令行环境？

A: 项目核心逻辑基于 Node.js 编写，可在 Windows PowerShell 或 CMD 中运行。但部分文件路径操作依赖于 posix 风格路径分隔符，建议在 Windows 下使用 Git Bash 或 WSL 以获得最佳体验。若在原生 CMD 中遇到路径解析问题，可将 `--data-dir` 参数中的反斜杠替换为双反斜杠或正斜杠。
