# LinkForge

LinkForge 是一个面向技术团队与内容运营者的轻量级外链资源聚合与导航系统。该项目定位于解决分散在各处的高价值技术文章、文档与工具链接难以统一管理、检索与共享的问题，尤其适用于需要频繁引用外部技术资料的中大型研发组织、技术社区运营者以及个人知识管理者。LinkForge 不依赖复杂的前端框架，以纯静态资源与后端路由结合的方式，提供高效的链接收录、分类索引与全文检索能力，帮助用户将零散的 URL 转化为结构化的知识资产。

## 功能概览

**结构化目录索引**：基于文章 ID 与分类标签生成多级索引，支持按技术领域、发布日期、热度等多维度筛选。

**全文元数据检索**：对每个收录的链接自动提取或手动补充标题、摘要、关键词，支持毫秒级本地检索响应。

**批量链接导入导出**：支持 CSV 与 JSON 格式的批量链接导入，便于从旧系统迁移或与团队共享链接集。

**访问状态监控**：定期对已收录链接进行 HTTP 状态检查，标记失效或重定向链接，保证资源可用性。

**自定义分类标签系统**：允许用户为每个链接添加多个自定义标签，构建符合自身知识体系的分组逻辑。

**响应式移动端适配**：前端界面针对手机与平板设备优化，确保在移动办公场景下的阅读与检索体验。

**RESTful API 支持**：提供完整的只读 API 接口，方便将链接数据集成到其他内部系统或自动化脚本中。

**用户收藏与备注**：支持登录用户对特定链接添加私人备注与收藏标记，实现个人化知识管理。

## 应用场景

技术团队内部知识库建设：研发团队可将日常开发中遇到的优秀技术博客、官方文档、调试工具链接统一收录至 LinkForge，按项目或技术栈分类，新成员入职时可直接通过该系统快速了解团队常用技术资源。

技术社区内容推荐管理：技术社区运营人员可使用 LinkForge 整理每周精选文章、视频教程和开源项目，生成可公开访问的资源导航页，减少重复推荐工作，提升内容分发效率。

个人开发者学习路径规划：个人开发者可将长期积累的学习资料、课程链接、代码示例按学习阶段（入门、进阶、实战）分类存放，配合备注功能记录学习心得，形成可迭代的个人知识库。

自动化链接健康度巡检：运维或质量保障团队可配置 LinkForge 的监控模块，定时扫描所依赖的外部文档链接是否有效，提前发现文档迁移或页面删除导致的链接断裂问题。

## 快速开始

以下命令可在 Ubuntu 22.04 或 macOS 12 及以上环境中完成 LinkForge 的本地部署。

```bash
# 克隆代码仓库
git clone https://github.com/your-org/linkforge.git

# 进入项目目录
cd linkforge

# 安装依赖（使用 pip 和 npm）
pip install -r requirements.txt
npm install --prefix frontend

# 初始化本地数据库与配置文件
python scripts/init_db.py
cp .env.example .env

# 启动开发服务器（后端 Flask + 前端 Vite 代理）
python run.py
```

执行上述步骤后，访问 http://localhost:5000 即可进入 LinkForge 首页。默认管理员账号为 admin，密码为 admin123，首次登录后请及时修改。

## 常见问题

**Q: 如何将现有的大量链接快速导入 LinkForge？**

A: 推荐使用 CSV 导入功能。您可以将链接、标题、标签按模板整理为 CSV 文件，然后通过管理后台的“批量导入”页面上传。系统会自动识别列头并进行数据校验。对于超过 5000 条的大批量导入，建议使用命令行脚本 `scripts/import_from_csv.py`，该脚本支持分批提交以避免内存溢出。

**Q: 链接健康度监控会影响页面访问速度吗？**

A: 不会。监控任务通过独立的后台进程执行（默认每日凌晨 2:00 触发），使用异步 HTTP 客户端并发检查，不阻塞前端请求。您也可以在配置文件中调整监控频率或完全禁用该功能。监控结果会缓存至 Redis，前端展示状态时直接读取缓存，响应时间控制在 50 毫秒以内。

**Q: 能否将 LinkForge 部署到内网环境且不连接互联网？**

A: 完全可以。LinkForge 的核心功能（链接管理、检索、分类）完全不依赖外部 CDN 或在线服务。前端资源在构建时已全部打包至本地，所有依赖库均随代码仓库分发。您只需确保 Python 和 Node.js 环境可用，安装依赖时需使用内网镜像源（如公司私有 PyPI 和 npm 仓库），之后即可完全离线运行。
