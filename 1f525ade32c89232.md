# LinkStash Navigator

LinkStash Navigator 是一个面向技术研究人员、开发者和内容策展人的轻量级外链资源汇总与导航系统。该项目专注于将分散于各类技术博客、新闻站点和文档中心的高质量文章链接进行结构化组织，并提供标签过滤、全文检索和访问频率统计等辅助功能，帮助用户在信息过载的环境中快速定位到真正有价值的技术内容。

本项目不生产原创内容，而是通过人工筛选与社区提交通道，建立一个经过质量校验的外部资源索引库。项目内置的链接健康检查模块会定期验证各外链的可访问性，并对失效链接进行自动标记和邮件通知，确保资源库的长期可用性。LinkStash Navigator 适合作为团队内部的知识库入口，也可部署为个人开发者的技术阅读工作台。


## 功能概览

**链接分类与标签系统** 对每一条外链按技术领域、内容类型和阅读时长进行多维度标签标注，支持自定义标签扩展。

**全文检索与模糊匹配** 基于标题、摘要、标签和来源站点的全文索引，支持拼音首字母缩写检索和拼写错误容错匹配。

**定期健康检查** 每日定时任务对所有存储的链接发起 HEAD 请求，检测状态码变化和响应时间波动，自动标记异常链接。

**访问统计与热门排序** 记录每条链接的点击次数和最近访问时间，提供按热度、新鲜度和随机三种排序模式。

**导入导出与备份** 支持 JSON、CSV 和 Markdown 表格三种格式的批量链接导入导出，每日自动生成增量备份文件。

**用户收藏与自定义分类** 允许注册用户建立个人收藏夹，并将链接拖拽归类到自定义的层级目录中，实现个性化资源管理。

**黑暗模式与阅读视图** 前端界面内置黑暗模式切换，外链详情页提供去噪阅读视图，过滤侧边栏和广告区域以提升阅读体验。

**开放 API 接口** 提供 RESTful API 用于链接的增删改查和批量操作，支持第三方工具集成和自动化脚本调用。


## 应用场景

**技术团队内部知识库构建** 团队技术负责人可以每日将团队遇到的疑难问题解决方案、第三方库踩坑记录、性能优化案例等高质量外链统一录入系统，并按照项目或微服务模块进行分类。新成员入职时可直接通过导航系统快速了解团队常用的技术栈参考文档和最佳实践链接，缩短上手周期。

**个人开发者每日阅读工作流** 开发者在日常浏览技术社区时，可以将有价值的文章链接一键存入 LinkStash Navigator，并标注预计阅读时间和技术标签。系统根据用户历史阅读偏好生成每日推荐列表，帮助开发者制定系统性的技术学习计划，避免碎片化阅读。

**技术文章聚合与 Newsletter 素材收集** 技术博主或社区运营人员可以将多个信息源的高质量文章汇总到 LinkStash Navigator 中，利用标签和分类功能快速筛选出某一主题下的多篇文章，作为周报或 Newsletter 的素材池。导出功能支持一键生成带注释的链接列表，直接粘贴到邮件内容或博客文章中。

**离线文档中心镜像** 在网络受限的内网开发环境中，运维人员可将外网的技术文档链接通过 LinkStash Navigator 的批量导入功能录入，并结合健康检查功能定期验证镜像站点的可用性。系统输出的健康报告可作为文档迁移和备份策略的决策依据。


## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，假设系统已安装 Git 和 Node.js 18.x 及以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/example/linkstash-navigator.git
cd linkstash-navigator

# 安装依赖包
npm install

# 配置环境变量，复制示例配置文件并修改数据库连接字符串
cp .env.example .env

# 初始化数据库表结构并导入预置链接索引
npm run db:migrate
npm run db:seed

# 以开发模式启动应用，默认监听 3000 端口
npm run dev
```

执行完毕后，在浏览器中访问 http://localhost:3000 即可打开 LinkStash Navigator 的主界面。首次启动会自动创建管理员账户，默认用户名和密码请查看 .env.example 文件中的注释说明。生产环境部署请参考 `docs/deployment.md` 文档，使用 PM2 或 Docker 进行进程管理。


## 安装要求

| 依赖项 | 必需版本 | 说明 |
|--------|----------|------|
| Node.js | 18.x 或 20.x LTS | 运行时环境，推荐使用 nvm 管理多版本。 |
| PostgreSQL | 14.x 或 15.x | 主数据库，用于存储链接元数据、用户信息和访问日志。 |
| Redis | 6.x 或 7.x | 缓存与会话存储，用于提升检索速度和分布式锁管理。 |
| Elasticsearch | 8.x | 全文检索引擎（可选），未安装时回退为 SQL LIKE 模糊匹配。 |
| PM2 | 5.x | 生产环境进程守护（可选），用于集群模式部署。 |
| Nginx | 1.22+ | 反向代理与静态资源服务（生产环境推荐）。 |

## 项目结构

```
linkstash-navigator/
├── apps/                                 # 多应用子目录
│   ├── web/                              # 主前端应用 (Next.js)
│   │   ├── pages/                        # 页面路由组件
│   │   │   ├── index.tsx                 # 首页：链接列表与检索入口
│   │   │   ├── detail/[id].tsx           # 链接详情页与阅读视图
│   │   │   └── dashboard/                # 用户仪表盘（收藏、分类管理）
│   │   └── components/                   # 可复用 UI 组件（LinkCard, TagFilter, SearchBar 等）
│   ├── api/                              # 后端 API 服务 (Express.js)
│   │   ├── routes/                       # RESTful 路由定义（links, users, tags, health）
│   │   ├── controllers/                  # 请求处理控制器
│   │   ├── services/                     # 业务逻辑层（链接管理、健康检查调度）
│   │   └── models/                       # 数据模型与 ORM 映射
│   └── worker/                           # 后台任务进程 (BullMQ)
│       ├── queues/                       # 队列定义（健康检查队列、导入导出队列）
│       └── processors/                   # 任务处理器（链接抓取、状态更新、邮件通知）
├── config/                               # 环境配置文件与 schema 校验
│   ├── default.json                      # 默认配置项（端口、超时时间、分页大小）
│   └── custom-environment-variables.json # 敏感变量映射（数据库密码、API 密钥）
├── data/                                 # 本地数据存储目录
│   ├── backups/                          # 每日自动备份的 JSON 文件
│   └── logs/                             # 应用日志（按日期分片）
├── docs/                                 # 完整文档体系
```

1. 外链资源提交：通过 GitHub Issue 模板提交新的外链推荐，需包含标题、原始 URL、所属技术分类和 50 字以内的摘要说明。项目维护者会在 48 小时内审核并决定是否纳入索引库。

2. 代码贡献流程：Fork 本仓库到个人账号，在 `develop` 分支基础上创建功能分支。提交代码前需运行 `npm run lint` 和 `npm run test` 确保代码风格和测试通过。完成开发后向 `develop` 分支发起 Pull Request，并关联相关的 Issue 编号。

3. 文档改进与翻译：欢迎对 `docs/` 目录下的各类文档进行勘误、补充和英文翻译。文档更新无需复杂的测试流程，但需保证 Markdown 格式规范，并遵守中文文案排版指北中的空格规则。

4. 链接健康检查协助：如果发现资源列表中的某个链接无法访问，可以通过页面上的「报告失效链接」按钮提交反馈。系统将自动记录反馈来源并纳入下一轮健康检查的优先级队列。

5. 社区讨论与建议：参与 Discussions 板块中的技术选型讨论、功能投票和新标签体系设计。活跃贡献者将被邀请加入项目的 GitHub Team，获得直接推送权限。


## 常见问题

**Q: 资源列表中的链接访问时返回 403 或 404 状态码，应该如何处理？**

A: LinkStash Navigator 的健康检查模块会在每日凌晨 2:00 自动检测所有链接。如果用户手动发现失效链接，可使用页面上的「报告失效链接」功能。项目维护者会在收到报告后的 24 小时内进行人工复核，若确认失效则从主列表中移除并记录到失效归档表中。用户也可以在自己的收藏夹中手动移除该链接。

**Q: 能否导入其他书签管理工具导出的 HTML 文件？**

A: 目前支持 Chrome 书签导出 HTML、Raindrop.io 的 CSV 导出和 Pocket 的 HTML 导出格式。在「导入」页面选择对应的源类型并上传文件，系统会自动解析标题、URL 和文件夹层级，并映射到本项目的标签和分类体系。如果解析失败，系统会提供详细的错误行号提示，并允许用户下载未解析成功的条目清单进行手动修正。

**Q: 搜索时为什么有些链接无法被检索到？**

A: 全文检索默认仅索引链接的标题、摘要和用户自定义标签。链接正文内容不会被抓取或索引，因为本项目定位为外链导航而非全文代理。如果希望提高某条链接的检索命中率，建议在摘要字段中填写更丰富的关键词描述。对于启用 Elasticsearch 的部署实例，检索结果会按照 TF-IDF 算法排序，并提供相关度评分供参考。
