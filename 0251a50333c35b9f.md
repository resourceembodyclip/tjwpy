# WebFront Resource Aggregator

WebFront Resource Aggregator 是一个面向 Web 前端开发者、技术文档撰写者以及开源项目维护者的外链资源整理与导航工具。该项目将散落在互联网各处的技术文章、教程、案例和文档按照统一的数据结构进行归集，并提供基于关键词和分类的快速检索能力。

该项目解决的核心问题是：技术资料分散、书签难以维护、知识回溯成本高。通过将所有外部链接统一存于项目仓库内的结构化数据文件中，开发者可以像使用本地数据库一样查询、筛选和导出技术资源清单，同时支持通过 CI 流程自动生成静态导航页面。

本项目定位为技术资源外链汇总站，不存储任何第三方文章内容，仅保存 URL、标题、摘要和分类元数据。项目适合需要长期维护技术知识库的团队和个人，也适合作为技术博客或文档站点的外部链接补充模块。

第 50/56 批次共收录 180 个资源链接，所有链接均已通过基础可达性校验并归类至相应技术主题下。

## 功能概览

**结构化数据存储**：所有外链以 JSON 格式存储于 /data 目录下，每条记录包含 URL、标题、来源站点、分类标签和收录时间戳，便于程序化读取和二次加工。

**分类筛选与标签系统**：为每个资源分配至少一个技术领域标签，例如 JavaScript、性能优化、移动端适配、安全策略等，支持多维度交叉筛选。

**静态站点生成**：内置基于 Node.js 的构建脚本，可将原始数据渲染为响应式 HTML 导航页面，输出至 /dist 目录，可直接部署至任何静态托管服务。

**命令行交互工具**：提供 CLI 模块用于新增、删除、搜索和导出资源记录，所有操作均通过命令行完成，适合自动化脚本集成。

**变更追踪与审计**：所有资源变更通过 Git 提交记录进行追踪，每次增删改操作均记录操作人、时间和变更原因，满足团队协作场景下的审计需求。

**格式兼容性**：支持导入和导出 CSV、Markdown 表格和 JSON 三种格式，便于与其他知识管理工具（如 Notion、Obsidian）进行数据交换。

**定期健康检查**：内置链接可用性检测脚本，可定期运行并生成失效链接报告，帮助维护者及时清理或更新失效资源。

## 应用场景

**技术团队内部知识库构建**：团队可将日常阅读到的优质技术文章通过 CLI 工具一键收录至项目仓库，所有成员共享同一资源池，避免重复检索和知识孤岛。配合 CI 流水线，每次合并请求可自动更新团队导航站。

**开源项目文档的外部链接管理**：开源项目维护者可将 README 或文档中引用的大量外部链接集中迁移至本项目的资源目录中，通过分类标签和检索功能替代冗长的行内链接列表，提升文档可读性和维护效率。

**技术博客的延伸阅读模块**：个人技术博主可使用该项目整理与每篇博文相关的延伸阅读链接，通过嵌入生成的导航页面或导出 Markdown 列表，为读者提供结构化的扩展阅读清单，增强博客的专业深度。

**技术培训与课程资料汇总**：培训机构或教育工作者可将课程中涉及的参考文档、视频链接和案例源码地址统一收录，按课时或主题分类，学员可通过搜索功能快速定位特定章节的参考资料，降低课程支持成本。

**个人技术阅读工作流管理**：个人开发者可将每日浏览的技术资讯、GitHub 仓库和 API 文档通过浏览器书签导出或手动录入的方式集中管理，配合健康检查功能定期清理失效链接，形成可持续维护的个人技术资料库。

## 快速开始

以下命令可在本地环境完成项目克隆、依赖安装和静态站点构建，所有操作均基于 POSIX 兼容终端。

```bash
git clone https://github.com/webfront-resource/aggregator.git
cd aggregator
npm install
npm run build
```

执行完成后，打开 /dist/index.html 文件即可浏览本地生成的资源导航页面。若需启动开发服务器并实时预览，可运行 `npm run dev`，默认监听 3000 端口。

## 常见问题

**问：收录外部链接是否存在版权或法律风险？**

答：本项目仅存储 URL 地址及基础的元数据信息（标题、分类、收录时间），不复制、转载或缓存任何第三方文章正文内容。所有链接均指向原始来源站点，用户点击后跳转至第三方页面。本项目作为导航工具，性质类似于浏览器书签或搜索引擎索引，不涉及对内容的复制与传播。若任何链接指向的内容存在侵权问题，请通过 GitHub Issues 联系我们，我们将依据相关权利方的要求及时移除该链接。

**问：如何批量导入现有书签或收藏夹中的大量链接？**

答：项目提供了 `scripts/parse-import.js` 脚本，支持从 CSV 文件和 Markdown 列表格式中批量导入。CSV 文件需包含 url、title、tag 三列。若使用浏览器导出的 HTML 书签文件，可先将书签转换为 CSV 格式（可使用在线转换工具或编写简单脚本），再执行 `node scripts/parse-import.js --input bookmarks.csv --format csv` 完成导入。导入前建议先备份 data/resources.json 文件，以防止意外覆盖。

**问：健康检查脚本如何工作？如何设置自动检测失效链接？**

答：健康检查脚本 `bin/health.js` 会遍历 data/resources.json 中的所有 URL，使用 Node.js 原生 http 模块发送 HEAD 请求，根据返回的 HTTP 状态码判定链接是否有效。状态码 200-299 视为有效，404 或超时视为失效。检测完成后在控制台输出失效链接列表，并生成一份 JSON 格式的报告文件存放在 /reports 目录下。如需设置自动检测，可在 .github/workflows/health-check.yml 中配置 cron 定时任务，默认每周一 00:00 运行一次，检测结果将通过 GitHub Actions 的 Summary 输出展示。
