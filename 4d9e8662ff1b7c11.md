# LinkVault Resource Aggregator

LinkVault is a curated technical resource aggregation platform designed for developers, researchers, and technical writers who need organized access to domain-specific reference materials, implementation guides, and troubleshooting documentation. The project systematically catalogs external technical articles, code examples, and architecture explanations from across the web, presenting them through a unified indexing interface.

This repository serves as a structured knowledge base that reduces the time spent searching for specialized technical content. Instead of maintaining bookmarks or scattered notes, users can query the indexed resource set through documented categories, track updates to external references, and contribute new entries following the project's curation standards. LinkVault targets teams building internal documentation portals, individual developers maintaining learning paths, and educators assembling course reading lists.

## 功能概览

**结构化资源索引** - Each external article is cataloged with metadata including source domain, content type, and estimated technical depth, enabling filtered browsing without leaving the repository interface.

**批量导入处理器** - Supports ingestion of large URL lists through a command-line interface that validates accessibility, extracts metadata, and generates markdown index entries automatically.

**分类标签系统** - Resources are tagged by programming language, framework, architecture pattern, and difficulty level, with support for multi-tag queries across the entire corpus.

**版本跟踪通知** - Monitors external URLs for content changes and notifies maintainers when referenced articles are updated or become inaccessible, preserving link integrity.

**全文检索接口** - Provides a lightweight search endpoint that scans resource titles, descriptions, and extracted keywords without requiring external search engine dependencies.

**导出模块** - Enables generation of resource lists in multiple formats including plain markdown, JSON for API consumption, and CSV for spreadsheet analysis.

**自定义分类规则** - Allows users to define regular expression patterns and keyword filters that automatically assign resources to user-defined categories during import.

**离线缓存预览** - Stores HTML snapshots of external articles for quick preview and offline reading, with configurable storage limits and expiration policies.

## 应用场景

**技术团队内部知识库构建** - Engineering teams can use LinkVault to aggregate external reference links relevant to their tech stack, adding internal annotations and difficulty ratings. The batch import feature allows onboarding existing bookmark collections, while the version tracking ensures team members always reference the latest documentation.

**开源项目文档增强** - Open source maintainers can embed LinkVault indexes within their project documentation to point contributors to related tutorials, API references, and community discussions. The structured format integrates with existing markdown documentation pipelines.

**技术课程资源管理** - Educators and bootcamp instructors can organize weekly reading lists, lab references, and supplementary materials through LinkVault's categorization system. The export feature generates student-facing resource handbooks in multiple formats.

**个人开发学习路径规划** - Developers can maintain a personal technical reading list spanning multiple domains. The search and filter capabilities help locate previously saved resources by topic, while the import processor handles bulk additions from saved browser bookmarks.

**合规性文档引用追踪** - Organizations with regulatory requirements can use LinkVault to track external standards documents, specification references, and compliance guidelines. The change notification system alerts teams when referenced external content is modified.

## 快速开始

```bash
# Clone the repository
git clone https://github.com/example/linkvault.git
cd linkvault

# Install dependencies
pip install -r requirements.txt

# Initialize the resource database
python linkvault.py init --db ./data/resources.db

# Import a batch of URLs from a text file
python linkvault.py import --source urls.txt --category "technical-articles"

# Start the local index server
python linkvault.py serve --port 8080 --host 127.0.0.1
```

## 贡献指南

1. 资源条目贡献: 通过提交 Issue 或在指定目录下创建符合格式规范的 markdown 文件来添加新的外部资源链接，每个条目必须包含 URL、简短描述和至少一个分类标签。提交前请运行 `python linkvault.py validate` 验证格式正确性。

2. 代码贡献流程: 从 main 分支创建功能分支，遵循 PEP 8 代码风格并使用 black 格式化。编写单元测试覆盖新增或修改的代码路径，确保所有测试通过后再发起 Pull Request。提交信息应使用约定式提交格式（如 feat:, fix:, docs:）。

3. 分类体系扩展: 如需增加新的顶级分类或修改现有分类定义，请先通过 Issue 讨论获得共识，然后更新 docs/categorization.md 中的分类表格，并同步修改代码中的分类验证逻辑。

## 常见问题

**Q: 批量导入大量 URL 时出现网络超时如何处理？**

A: 导入器默认超时设置为 30 秒。对于大量 URL 导入，建议使用 `--timeout 60` 参数增加超时阈值，或使用 `--retry 3` 启用自动重试机制。如果特定域名持续超时，可以将其加入 `config/ignore_domains.txt` 跳过检查，仅记录元数据。网络环境不稳定的情况下，建议先使用 `--dry-run` 模式验证 URL 可达性再执行实际导入。

**Q: 如何迁移数据库到另一台机器？**

A: 直接复制 `data/resources.db` 文件到目标机器的相同相对路径即可完成迁移。若需要跨 SQLite 版本迁移，请先运行 `scripts/migrate_db.py --export` 导出为 SQL 转储文件，然后在目标机器上运行 `scripts/migrate_db.py --import` 重建数据库。缓存目录 `data/cache/` 为可选迁移内容，若不复制则服务启动时会自动重新获取外部内容。

**Q: 离线缓存占用的磁盘空间如何控制？**

A: 在 `.env` 文件中设置 `CACHE_MAX_SIZE_MB=1024` 限制缓存总大小（单位 MB）。系统每天凌晨执行一次自动清理，按最后访问时间删除最旧的缓存条目直至低于限制值。也可以手动运行 `python linkvault.py cache --prune 500` 立即将缓存压缩到指定大小（单位 MB）。缓存文件存储在 `data/cache/` 目录下，可以直接删除该目录重置缓存。
